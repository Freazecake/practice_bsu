# Учёт товаров на складе (Qt, Model–View)

Практическое задание №1 по Qt: приложение для отображения и редактирования табличных
данных с архитектурой **Model–View**. Вариант: **учёт товаров на складе**.

## Сборка и запуск

### Вариант A — Qt Creator
1. Откройте `WarehouseApp.pro` в Qt Creator (Qt 5.15+ или Qt 6.x).
2. Нажмите **Собрать и запустить** (Ctrl+R).

### Вариант B — CMake из терминала
```bash
mkdir build && cd build
cmake .. -DCMAKE_PREFIX_PATH=<путь_к_Qt>   # например /opt/Qt/6.7.0/gcc_64
cmake --build .
./WarehouseApp
```

### Вариант C — qmake из терминала
```bash
qmake WarehouseApp.pro
make
./WarehouseApp
```

## Структура проекта

| Файл | Назначение |
|---|---|
| `warehousemodel.h/.cpp` | Собственная модель данных на основе `QAbstractTableModel` |
| `itemdialog.h/.cpp` | Диалог добавления/редактирования товара с проверкой ввода |
| `mainwindow.h/.cpp` | Главное окно: таблица, кнопки, поиск, фильтр, сохранение/загрузка |
| `main.cpp` | Точка входа |

## Как реализованы требования задания

- **QTableView + модель** — `QTableView` отображает данные через собственную модель
  `WarehouseModel : public QAbstractTableModel` (не `QStandardItemModel`).
- **5 столбцов, 10 записей** — ID, Наименование, Категория, Количество, Цена;
  при старте загружается 10 демонстрационных товаров.
- **Добавление / редактирование / удаление** — кнопки "Добавить", "Редактировать",
  "Удалить" (двойной клик по строке тоже открывает редактирование).
- **Ввод/вывод данных** — через диалоговое окно `ItemDialog` с валидацией.
- **Сохранение и загрузка в 2 форматах** — `.txt` (поля разделены `|`) и `.csv`
  (стандартный CSV с экранированием кавычек).
- **Обработка ошибок** — при некорректном формате файла, отсутствии прав на запись
  и т.п. выводится `QMessageBox::critical` с описанием ошибки; при пустом
  наименовании или отрицательных числах ввод отклоняется.

## Дополнительные возможности

- **Поиск** — строка поиска фильтрует таблицу по наименованию в реальном времени
  (`QSortFilterProxyModel`).
- **Фильтрация** — выпадающий список категорий сужает список товаров.
- **Сортировка** — клик по заголовку столбца сортирует таблицу (по возрастанию/убыванию).
- **Проверка корректности ввода** — пустое имя, отрицательное количество/цена
  отклоняются с предупреждением.
- **Подсветка строк** — товары с остатком ≤ 10 шт. подсвечиваются красным
  (низкий остаток на складе).
- **Статистика** — строка состояния показывает количество записей, суммарное
  количество товара, общую стоимость склада и число позиций с низким остатком.

## Формат файлов

**CSV** (`warehouse.csv`):
```
ID,Наименование,Категория,Количество,Цена
1,Ноутбук,Электроника,15,45000.00
```

**TXT** (`warehouse.txt`, разделитель `|`):
```
1|Ноутбук|Электроника|15|45000.00
```
