# Карточка товара (Qt, SDI-приложение)

Практическое задание №3 по Qt: SDI-приложение для ввода, редактирования,
сохранения и загрузки текстовой и числовой информации в текстовом
(`QTextStream`) и бинарном (`QDataStream`) форматах. Вариант: **карточка товара**.

## Сборка и запуск

### Qt Creator
Откройте `ProductCardApp.pro` (Qt 5.15+ / Qt 6.x) и нажмите **Run**.

### CMake
```bash
mkdir build && cd build
cmake .. -DCMAKE_PREFIX_PATH=<путь_к_Qt>
cmake --build .
./ProductCardApp
```

### qmake
```bash
qmake ProductCardApp.pro && make && ./ProductCardApp
```

## Структура проекта

| Файл | Назначение |
|---|---|
| `productcard.h/.cpp` | Структура записи `ProductCard` + операторы `QDataStream` для бинарной сериализации |
| `productcardstore.h/.cpp` | Бизнес-логика (CRUD, поиск, сортировка, валидация, сохранение/загрузка) — отделена от GUI, поэтому тестируется без `QApplication` |
| `mainwindow.h/.cpp` | SDI-окно: меню, список записей, форма редактирования |
| `main.cpp` | Точка входа |
| `tests/` | Модульные тесты (QtTest), отдельный исполняемый файл |
| `sample_data/` | Тестовые данные — `products.txt` и `products.dat`, сгенерированные приложением |

## Как реализованы требования задания

- **SDI, одно главное окно** — `MainWindow : public QMainWindow`.
- **Меню** — Файл (Создать / Открыть текст / Открыть бинарный / Сохранить (оба формата) / Выход),
  Правка (Добавить / Удалить / Найти), Справка (О программе).
- **Поля записи**: 2 текстовых (Наименование, Категория), 2 числовых (Количество — `QSpinBox`,
  Цена — `QDoubleSpinBox`), 1 дополнительное (Дата поступления — `QDateEdit`).
- **Ввод/редактирование** — форма справа; кнопка «Применить изменения» валидирует и сохраняет
  правки в выбранную запись.
- **Сохранение/загрузка в двух форматах**:
  - текстовый — `QTextStream`, разделитель `|`, поле экранируется при наличии `|`;
  - бинарный — `QDataStream` с заголовком (magic number + версия формата), что позволяет
    отличить чужой/повреждённый файл от валидного.
- **Обработка ошибок** — все операции с файлами оборачивают `QFile`/`QDataStream::status()`
  и выводят `QMessageBox::critical` с понятным описанием проблемы.

## Дополнительные требования (реализовано 5 из 6)

- **Валидация данных** — `ProductCardStore::validate()`: обязательные текстовые поля не пустые,
  количество и цена — в допустимом диапазоне, дата — валидна.
- **Поиск данных** — `findByText()` ищет по названию, категории и ID (регистронезависимо).
- **Сортировка данных** — `sortBy()` сортирует по любому из 6 полей, по возрастанию/убыванию.
- **QSettings** — сохраняются геометрия окна и пути к последним открытым файлам
  между запусками (`QSettings("QtLab3", "ProductCardApp")`).
- **Подтверждение действий** — удаление записи и закрытие/переключение файла при
  несохранённых изменениях запрашивают подтверждение через `QMessageBox`.

## Модульные тесты (QtTest)

`tests/tst_productcardstore.cpp` — отдельный класс `TestProductCardStore`,
собирается в самостоятельный исполняемый файл `ProductCardStoreTests`
(не зависит от GUI). Реализовано 11 тестов:

| Требование | Тесты |
|---|---|
| Создание/изменение данных | `testAddRecord`, `testUpdateRecord`, `testRemoveRecord` |
| Сохранение/загрузка файлов | `testSaveAndLoadText`, `testSaveAndLoadBinary` |
| Обработка ошибок | `testValidationRejectsInvalidData`, `testLoadMissingFileFails`, `testLoadCorruptedTextFails`, `testLoadCorruptedBinaryFails` |
| Доп. функциональность | `testFindByText`, `testSortByField` |

Запуск:
```bash
cd build
ctest --output-on-failure          # или напрямую: ./tests/ProductCardStoreTests
```

Результат: **13 passed, 0 failed** (11 тестов + `initTestCase`/`cleanupTestCase`).
